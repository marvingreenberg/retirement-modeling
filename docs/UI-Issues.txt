# UI Refactoring Issues Log
# Each section documents issues, decisions, and warnings per change.

## 1. ux-first-use-flow

- Decision: defaultPortfolio changed to empty state (age 0, no accounts, $0 spend).
  Tests that relied on old defaults switched to samplePortfolio.
- Decision: Setup age validation uses 20-120 range (not 0-120) since nobody
  under 20 is planning retirement.
- Issue: spending.test.ts failed after switching to samplePortfolio because
  samplePortfolio uses 'guardrails' strategy but tests expected 'fixed_dollar'.
  Resolution: beforeEach now explicitly sets spending_strategy to 'fixed_dollar'.
- Issue: First full test run timed out (resource contention with 13 test files).
  Second run passed cleanly. Likely machine load, not a code issue.
- No other issues.

## 2. ux-income-expansion

- Decision: IncomeEditor now accepts full `config` prop (not just socialSecurity)
  to access ss_auto, income_streams, and current_age_spouse for spouse visibility.
- Decision: ss_auto syncs values to legacy social_security via $effect for backward
  compat — older portfolio files still work.
- Decision: Zod schema changed annual_spend_net from .positive() to .min(0) to
  allow empty default portfolio (annual_spend_net: 0).
- Issue: IncomeEditor tests initially tried to test add/remove via PortfolioEditor
  with CollapsibleSection expansion. Section expansion didn't work reliably in jsdom.
  Resolution: Test IncomeEditor directly with config prop for read-only checks.
- No other issues.

## 3. ux-details-page

- Decision: Added shared simulationResults store to make results accessible from
  /details page. Loading/error state remains local to +page.svelte.
- Decision: Removed CollapsibleSection year-by-year table from SimulateView entirely.
  Details page now owns the full table for single-run and MC percentile table.
- Fix: api.test.ts was missing income_streams and ss_auto fields (pre-existing from
  ux-income-expansion). Fixed while here.
- Issue: navigation.test.ts dynamic import of details page hangs when lucide-svelte
  icons are involved. The old placeholder had no script block so it worked. Resolution:
  Removed the smoke test from navigation.test.ts (details page has its own test file
  with 4 tests covering empty state, single-run table, MC table, and heading).
- No other issues.

## 4. ux-config-drawer

- Decision: Used inline "Advanced" text toggle within SimulateSettings rather than a
  separate drawer component. Simpler and keeps everything in one place.
- Decision: Tax rates, RMD age, and IRMAA limit moved behind "Advanced" toggle.
  Primary row keeps inflation, growth, and conversion strategy.
- No issues.

## 5. ux-landing-redesign

- Decision: WelcomeState now reads from portfolio store directly to show portfolio
  summary (total balance, annual spending, coverage estimate). No prop needed.
- Decision: Warning banner only shows when accounts exist but spending is $0.
  Kept deliberately simple — just one warning for the most common issue.
- Decision: Used CSS spinner (border-based animation) rather than a library component
  to avoid additional dependency.
- Decision: "View Details" links use different text for single-run vs MC results to
  set correct expectations about what the details page shows.
- No issues.
